# Pillyze Inspired Component Map

## AppHeader

- 얇은 모바일 헤더로 화면 제목과 보조 상태만 보여준다.
- 홈, 상담방, 저장, 내 정보, 분석권 화면에 공통 적용한다.
- 큰 앱바보다 앱 내부 화면처럼 보이는 밀도를 우선한다.

## CardGroup

- 흰색 surface 카드에 관련 정보를 묶는다.
- 온보딩 기본 정보, 조언 방식, 상담방 상태, 저장 원칙, 분석권 패키지에 사용한다.
- 카드 내부는 제목, 짧은 설명, 필드/칩/리스트 순서로 정리한다.

## CompactListTile

- 상담방, 저장 답장, 내 정보 설정을 같은 리스트 언어로 통일한다.
- 구성은 작은 원형 아이콘, 제목, 보조 설명, 상태 chip, chevron이다.
- 한 항목은 72–88px 높이를 기준으로 하고, 모바일에서 4개 정도가 한 화면에 보여야 한다.

## BottomCTA

- 사용자가 다음에 해야 할 행동이 명확한 화면에만 사용한다.
- 시작, 저장, 상담방 만들기, 분석 시작, 히스토리 저장, 분석권 충전에 사용한다.
- 보조 행동은 텍스트 버튼이나 chip으로 낮춘다.

## StatusChip

- 조언 수위, 관계 단계, 전략, 저장 상태, 분석 상태를 pill로 표시한다.
- 선택 상태는 rose accent, 정보/주의 상태는 semantic 색상만 사용한다.
- 칩 문구는 2–8자 내외로 짧게 유지한다.

## InsightGrid

- 분석 결과의 보조 정보를 2열 카드로 분리한다.
- `피해야 할 말`, `다음 행동`, `현재 상태`, `전략`처럼 답장 선택을 돕는 정보에만 사용한다.
- 1순위 답장 카드는 별도 큰 카드로 두고 그리드에 섞지 않는다.

## FormSelectField

- 새 상담방과 온보딩에서 예시 선택과 직접입력을 결합한다.
- 사용자는 예시를 선택한 뒤 문장을 수정할 수 있어야 한다.
- 실명 대신 별칭, 대화 원문 최소 저장, 압박 금지 안내를 함께 노출한다.
