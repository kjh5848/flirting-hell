## 1. 실무 운영: 퍼널 분석 및 개선

5주차 첫 번째 핵심은 서비스를 만든 뒤 데이터를 보고 개선하는 방법이다.

과거 방식은 데이터를 모으고, 분석하고, 가설을 세우고, 검증하는 순서였다. 현재는 AI 도구를 활용해 퍼널 데이터를 더 빠르게 요약하고, 문제 지점을 찾고, 개선 아이디어를 뽑는 방향으로 바뀌고 있다.

자료에서는 퍼널을 단계별로 보고, 사용자가 어디에서 이탈하는지 확인하는 관점을 강조한다.

예시 퍼널 이벤트:

- `landing_page_view`
- `click_get_started`
- `view_input_form`
- `upload_photo`
- `begin_checkout`
- `purchase`

핵심은 기능을 많이 만드는 것이 아니라, 사용자가 다음 단계로 넘어가는지 숫자로 확인하는 것이다.

### 플러팅지옥 적용

플러팅지옥은 다음 퍼널을 우선 측정한다.

1. 첫 화면 방문
2. 메시지 입력 시작
3. 분석 시작
4. 분석 결과 도달
5. 답장 복사
6. 재생성 클릭
7. 무료 한도 도달
8. 분석권 구매 진입
9. 분석권 결제 완료

이 퍼널은 `monetization-metrics.md`의 이벤트 이름과 연결한다.

## 2. 퍼포먼스 마케팅

두 번째 핵심은 퍼포먼스 마케팅이다.

브랜딩 광고는 노출, 인지도, 이미지 중심이다. 반면 퍼포먼스 마케팅은 돈을 넣었을 때 결과가 숫자로 바로 증명되는 마케팅이다.

자료에 나온 대표 채널:

- Google Ads
- Meta Ads
- TikTok Ads

자료에서는 Meta Business, Meta Pixel, 광고 성과 화면, 광고 소재 예시, 인스타그램 릴스/숏폼 사례를 보여준다.

중요 지표:

- ROAS: 광고비 대비 매출
- CTR: 노출 대비 클릭률
- CPC: 클릭당 비용
- CPA: 전환당 비용
- CPM: 1,000회 노출당 비용

핵심은 광고비를 쓰는 것이 아니라, 어떤 소재와 랜딩이 실제 행동으로 이어지는지 확인하는 것이다.

### 플러팅지옥 적용

초기에는 광고 집행보다 자연 유입과 테스트 유저 확보가 먼저다.

광고를 한다면 다음 소재가 적합하다.

- `답장 뭐 보내야 하지?` 상황 중심 숏폼
- 카톡/DM 대화 예시 기반 전후 비교
- `무한 플러팅 말고, 딱 맞는 한마디` 메시지
- 무료 3회 분석 체험 유도

광고 집행 전에는 반드시 이벤트 로그와 전환 추적을 먼저 준비한다.

## 3. 앱 개발

세 번째 핵심은 앱 개발 방식 비교다.

자료에서는 네이티브 앱, 크로스 플랫폼, 하이브리드 앱, PWA를 비교한다.

### 네이티브 앱

- iOS와 Android 각각 OS에 맞게 개발
- 성능과 기능 접근성이 가장 좋음
- 개발 속도가 느리고 필요한 지식이 많음

### 크로스 플랫폼

- 하나의 언어와 프레임워크로 iOS/Android를 함께 개발
- React Native, Flutter, Expo 같은 선택지가 있음
- 빠르게 양쪽 앱을 만들 수 있지만, 네이티브 대비 제한이 있음

### 하이브리드 앱

- 웹앱을 앱 껍데기 안의 WebView로 감싼 방식
- Capacitor 같은 도구가 해당
- 빠르게 앱처럼 만들 수 있지만 성능과 네이티브 기능은 제한됨

### PWA

- 모바일 앱처럼 보이게 만든 웹
- 홈 화면 추가로 앱 아이콘처럼 사용할 수 있음
- 앱스토어 출시나 네이티브 기능 접근은 제한됨

자료에서 언급된 도구:

- Capacitor: https://capacitorjs.com/
- Expo: https://expo.dev/
- Apple App Store Review Guidelines

### 플러팅지옥 적용

현재 결정한 `Cloudflare Pages + Workers + D1` 기반 모바일 웹/PWA 방향은 5주차 내용과 맞다.

V1은 PWA로 시작한다.

앱 전환은 다음 조건이 충족될 때 검토한다.

- 7일 재사용률이 충분히 높음
- 홈 화면 추가보다 푸시 알림이 중요해짐
- 앱스토어 검색 유입이 필요해짐
- 결제와 로그인 흐름이 안정됨

앱 전환 후보:

- 빠른 앱 포장: Capacitor
- 본격 앱 개발: Expo 또는 React Native

## 4. 미국 법인과 Stripe

네 번째 핵심은 미국 법인, Stripe, 글로벌 결제, 법인 주소, 전화번호, 은행 계좌다.

자료에서는 Stripe Atlas와 미국 Delaware 법인을 다룬다.

핵심 메시지:

- 미국 Delaware 법인은 투자, 인수, IPO가 굴러가는 글로벌 표준 레일에 올라타는 것
- 글로벌 확장과 Exit 기대값에 강함
- Stripe, Mercury, 투자자, VC 등 글로벌 생태계에서 더 익숙한 구조

자료에서 언급된 항목:

- Stripe Atlas
- Stripe 사용 이유
- 한국 결제와 Stripe/Polar 비교
- Polar acceptable use
- Delaware C-Corp
- LLC와 C-Corp 비교
- Registered Agent
- 미국 법인 주소
- 미국 전화번호
- 미국 은행 계좌
- Mercury

### 플러팅지옥 적용

지금 당장 미국 법인을 만들 필요는 없다.

현재는 Polar를 결제 제공자로 쓰기로 결정했으므로, V1에서는 다음만 확인하면 된다.

- Polar 상품 생성 가능 여부
- Polar acceptable use 위반 여부
- 원화 또는 달러 결제 표시 방식
- webhook으로 분석권 지급 가능 여부
- 환불 시 분석권 회수 정책

미국 법인과 Stripe Atlas는 다음 단계에서 검토한다.

- 글로벌 유료 매출이 실제로 발생
- 해외 결제 비중이 커짐
- 투자 또는 인수 가능성을 고려
- Stripe가 꼭 필요한 기능이 생김

## 5. Exit 전략

다섯 번째 핵심은 서비스를 만든 뒤 창업자가 선택할 수 있는 큰 경로다.

자료에서는 Exit 전략을 크게 세 가지로 나눈다.

1. 현금 흐름 유지
2. 매각
3. IPO

### 현금 흐름 유지

서비스가 안정적으로 돈을 벌면 매각이나 IPO 없이 계속 운영할 수 있다.

개인사업자는 소득으로 가져가고, 법인사업자는 급여나 배당 구조로 가져간다.

### 매각

매각은 하나의 방식만 있는 것이 아니다.

- 회사 통째로 팔기
- 회사 일부만 팔기
- 팀이나 사람 중심으로 팔기
- 사업 또는 자산만 팔기

자료에서 언급된 마켓플레이스:

- Acquire.com
- Flippa

### IPO

IPO는 가장 큰 규모의 Exit다.

자료에서는 나스닥, 쿠팡, OpenAI 관련 사례, 국내외 상장 사례 등을 보여준다.

핵심은 IPO 자체보다, 작은 서비스도 지표와 성장성이 있으면 더 큰 자본시장 또는 인수 시장과 연결될 수 있다는 관점이다.

### 플러팅지옥 적용

플러팅지옥은 초기부터 Exit를 목표로 기능을 키우기보다, 다음 순서가 현실적이다.

1. 메시지 분석 품질 검증
2. 답장 복사율 확인
3. 7일 재사용률 확인
4. 분석권 반복 구매 확인
5. 유료 사용자 인터뷰
6. 카테고리 확장 또는 앱 전환 검토

Exit 관점에서 중요한 자산은 코드보다 데이터와 브랜드다.

- 익명화된 대화 상황 유형
- 말투 프로필 패턴
- 이상형/연애 스타일 적합도 모델
- 답장 복사율과 유료 전환율
- `플러팅지옥`이라는 기억나는 브랜드

## 플러팅지옥 MVP에 바로 반영할 결정

### 유지할 결정

- V1은 모바일 웹/PWA로 시작한다.
- 앱스토어 출시는 후순위다.
- Cloudflare Pages, Workers, D1 조합은 적절하다.
- Polar 결제는 3주차와 5주차 강의 흐름 모두와 맞는다.
- 분석권 패키지는 초기 과금 방식으로 적합하다.

### 추가해야 할 작업

- 퍼널 이벤트를 코드에 먼저 심는다.
- 무료 3회 분석과 분석권 잔액을 D1에 저장한다.
- Polar webhook 중복 처리를 반드시 구현한다.
- PWA 설치 가능성을 초기부터 고려한다.
- 앱 전환은 Capacitor와 Expo 중 하나를 나중에 비교한다.

### 보류할 작업

- 네이티브 앱 출시
- 미국 법인 설립
- Stripe Atlas
- IPO/매각 전략 실행
- 대규모 광고 집행

## 개발 우선순위

1. Vite React 앱 생성
2. Cloudflare Pages 배포
3. Workers API 생성
4. D1 데이터 모델 작성
5. AI 분석 API 구현
6. 이벤트 로그 저장
7. 무료 분석 횟수 제한
8. Polar sandbox 결제 연동
9. webhook 기반 분석권 지급
10. PWA 설정

## 한 줄 요약

5주차 자료의 핵심은 `만든 서비스를 데이터로 운영하고, 돈을 써서 성장시키고, 웹을 앱으로 확장하고, 글로벌 법인/결제/Exit까지 갈 수 있는 로드맵을 이해하라`는 것이다.

플러팅지옥은 이 중에서 지금 당장 `퍼널 분석`, `PWA`, `Polar 결제`, `분석권 잔액 관리`까지만 MVP에 반영하면 된다.
