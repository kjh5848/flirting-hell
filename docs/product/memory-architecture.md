# 메모리·대화 연속성 아키텍처

> 목적: 사용자가 답장을 받고 → 보내고 → 다음 대화를 이어 붙일 때, 코치가 **이전 흐름을 기억하고 이어서** 코칭하게 한다. 세션이 길어져도 비용·품질·프라이버시를 통제한다.

## 0. 핵심 원칙

- **1M 컨텍스트 ≠ 메모리.** 한 번의 호출 창일 뿐이고, 입력 토큰은 매 호출 과금된다. 히스토리를 매번 다 넣으면 호출당 비용이 계속 커지고 "lost in the middle" 품질 저하가 생긴다. → 1M은 **천장(안전장치)**이지 전략이 아니다.
- **원문 미저장.** 메모리는 항상 **파생물**(구조화 상태·요약)이며 대화 원문이 아니다([AGENTS.md](../../AGENTS.md) 프라이버시 규칙).
- **요약은 프로즈 블롭이 아니라 구조화 상태로.** 프로즈를 매 턴 다시 쓰면 누적 손실(드리프트)이 생긴다. → 사실(facts)은 append-only, 서사는 가볍게 재생성.

## 1. 4계층 메모리

| 계층 | 내용 | 저장 |
| --- | --- | --- |
| 단기 | 이번에 붙여넣은 **원문** | LLM에 보내고 버림 |
| 중기 | 상담방의 **이전 turn 요약**(상태·상대유형·추천) | ✅ 이미 저장(`analysis_turns`) |
| 롤링 | 오래된 turn을 **구조화 관계상태**로 압축 | Phase A.5/B |
| 장기 | 내 성향·이상형 | ✅ 영속(`user_profiles`) |

## 2. 구조화 관계상태 (facts vs narrative)

- **facts (append-only, 거의 무손실)**: 관계 단계, **최신 상대 5축 유형**, 이전에 하기로 한 약속/날짜, 상대 선호, 주의신호.
- **narrative (재생성, 손실 허용)**: "지금 분위기" 한두 줄. 가벼우니 매번 새로 만들어도 됨.
- **최근 K턴**: 요약 원본 그대로(recency). 상담방에 이미 있음.

## 3. 호출 시 컨텍스트 조립

```
[내 성향/이상형]
+ [구조화 관계상태(facts)]
+ [최근 K턴 요약]
+ [(오래되면) RAG로 검색한 관련 턴 top-N]
+ [이번 원문]
→ "이전 흐름을 이어서, 내 성향과 상대 유형 차이를 반영해 코칭하라"
```

→ 히스토리가 길어도 **보내는 양이 일정** = 비용·품질 통제.

## 4. 단계적 도입

- **Phase A (지금 구현)**: 분석 시 `내 성향 + 최근 K턴 요약 + 최신 상대유형`을 `AnalysisRequest`에 주입 → mock/LLM이 **이어서 코칭 + 답장 개인화**(DoD #4). 새 DB 컬럼·임베딩 불필요(기존 turn 재사용).
- **Phase A.5**: 상담방에 구조화 관계상태(rollup) 컬럼 추가 + turn이 늘면 compaction.
- **Phase B**: 관계가 수개월·수백 턴이면 **pgvector**(Postgres 확장)로 turn 요약 RAG 추가. 임베딩은 **요약 기준**(원문 아님 → 프라이버시 유지).
- **1M**: 드물게 컨텍스트 넘칠 때 천장으로만.

## 5. 요약 품질 가이드

- 프로즈 롤링 요약 **단독 금지**(드리프트). 사실은 구조화 필드로 append/갱신.
- 상대 5축 유형은 **최신값 유지**(또는 신뢰도 가중 갱신), 매 턴 덮어쓰지 말고 변화만 반영.
- 요약/임베딩은 항상 파생물 기준 — 원문은 단기 계층에서만 쓰고 버린다.

## 6. Phase A 구현 메모 (이 분기)

- `AnalysisPort.AnalysisRequest`에 `myPersonalitySelf`, `myPersonalityIdeal`, `recentSummaries`(최근 turn 요약 리스트), `latestPartnerType` 추가.
- `AnalysisService.createAnalysis`가 bootstrap 프로필(성향)과 상담방 최근 turn에서 이 맥락을 조립해 주입.
- `MockAnalysisAdapter`: 내 성향↔상대 유형 차이로 답장/다음행동/주의를 조정하고, 최근 요약이 있으면 "이어서" 톤으로.
- `LlmAnalysisAdapter`: 같은 맥락을 userPrompt에 포함(스키마 변경 없음).
- 전부 mock·테스트로 검증 가능. 실 LLM은 키 투입 시 동작.
