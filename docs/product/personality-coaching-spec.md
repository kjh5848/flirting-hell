## 0. 한 줄 정의

사용자의 **연애 성향**을 입력받아 → ① 내가 원하는 **이상형과의 적합도**, ② 대화로 추론한 **상대 유형**을 판별해 → 두 정보를 토대로 "이 관계가 맞는지 / 어떻게 다가갈지"를 코칭한다. 단순 답장 생성기가 아니라 **성향·궁합 진단형 코치**.

## 0-1. 최종 목표 (North Star) — 사용자 승인 2026-06-16

사용자가 **내 성향 + 이상형**을 계정에 저장해두면, 각 상담방에서 대화를 붙여넣을 때:

1. AI가 **상대의 5축 유형**을 추론한다(근거는 요약만 저장, 원문 미저장).
2. **이상형 적합도(%)**를 결정적으로 계산해 보여준다.
3. **내 성향 ↔ 상대 유형 차이**를 근거로 답장·다음 행동·플랜을 맞춤 코칭한다.
4. 결과 화면: `상대 유형 카드 + 적합도 + 축별 갭 + 그에 근거한 코칭`.

이후 챗형 코칭과 구체 플랜(데이트 코스·확인 포인트)이 이 성향 레이어 위에 얹힌다.
**"내가 어떤 사람이고, 상대가 어떤 유형이며, 우리가 맞는지 + 어떻게 다가갈지"를 한 흐름에서 답하는 코치**가 완성 상태다.

## 1. 기존 제품 위에 어떻게 얹히나

- 지금: 상담방 → 대화 붙여넣기 → 분석(요약·답장·전략). (단발 구조화 출력)
- 추가: **성향 레이어**가 분석 위에 얹힌다.
  - 내 성향/이상형 = 전역 프로필(온보딩·내 정보에서 설정).
  - 상대 유형 = 상담방의 대화 분석에서 추론(LLM).
  - 궁합/적합도 = 내 이상형 ↔ 상대 유형 비교(결정적 계산).
  - 코칭 = 내 성향 ↔ 상대 유형 차이를 근거로 답장·플랜에 반영.

## 2. 성격 프레임워크 — ★결정 필요★

### ✅ 확정(사용자 승인 2026-06-16): 자체 5축 "연애 성향"

| 축 ID | 낮음(1) ↔ 높음(5) | 의미 |
| --- | --- | --- |
| `expression` | 신중함 ↔ 적극적 | 호감 표현을 얼마나 먼저·세게 하나 |
| `pace` | 천천히 ↔ 빠르게 | 관계 진전 속도 선호 |
| `contact` | 가끔 ↔ 자주 | 연락 빈도 선호 |
| `emotion` | 은근함 ↔ 직설적 | 감정 표현 방식 |
| `values` | 현실적 ↔ 낭만적 | 연애 가치관 |

**근거**
- 연애 맥락에 직접 맞닿아 코칭으로 바로 연결된다(MBTI의 추상 축보다 실용적).
- 5축 1~5점 = 입력이 가볍고(슬라이더 5개), 대화에서 LLM이 추정하기 쉽다.
- 라이선스/타당성 리스크 없음(MBTI는 상표·심리측정 논란).
- 적합도 = 두 5차원 벡터의 거리로 **결정적 계산** 가능 → 테스트 가능.

### 대안 (깨서 바꾸고 싶을 때)
- **MBTI식 16유형**: 인지도 높고 마케팅 친화적. 단 라이선스·타당성 논란, 16유형은 입력/추론 부담 큼.
- **빅5(OCEAN)**: 학술적 타당성 최고. 단 연애 맥락과 거리가 있어 코칭 번역이 한 단계 더 필요.
- **혼합**: 표면은 친근한 "연애 유형 4~6종" 라벨(마케팅), 내부는 5축 점수(엔진). → 장기적으로 가장 강력. 추천 5축을 엔진으로 두면 이 길로 확장 쉬움.

> 교체 비용: 축 정의는 `apps/mobile/lib/features/personality/domain/personality_axes.dart` 한 곳. 프레임워크를 바꾸면 이 파일 + 적합도 계산만 손대면 된다.

## 3. 데이터 스키마 (개념)

```text
PersonalityProfile        # 전역 프로필에 추가
  self:  Map<axisId, 1..5>   # 내 성향
  ideal: Map<axisId, 1..5>   # 내 이상형(원하는 상대)

PartnerType               # 상담방별, 대화에서 LLM 추론 (이번 밤 범위 밖)
  axes: Map<axisId, 1..5>
  confidence: 0..1
  evidence: String           # 추론 근거(원문 미저장 원칙 준수: 요약만)

Compatibility             # 결정적 계산
  score: 0..100              # distance(ideal, partner) → %
  perAxis: Map<axisId, gap>  # 축별 차이
  highlights: 잘 맞는/주의 축
```

## 4. 흐름

1. 온보딩/내 정보에서 **내 성향 + 이상형** 설정(슬라이더 5개씩).
2. 상담방에서 대화 분석 시 **상대 유형 추론**(분석 출력에 axes 추가). ← *LLM, 이번 밤 범위 밖*
3. **적합도 카드**: 이상형 ↔ 상대 유형 거리로 점수 + 축별 갭 + 코칭 포인트.
4. 코칭/답장: 내 성향과 상대 유형 차이를 프롬프트에 반영.
