# 플러팅지옥 무료/유료 측정값

## 목적

이 문서는 플러팅지옥 V1에서 무료 기능과 유료 기능을 어떻게 나누고, 어떤 지표로 전환 가능성을 판단할지 정의한다.

무료는 사용자가 실제 가치를 경험할 수 있어야 한다. 유료는 더 많은 분석, 더 깊은 개인화, 더 긴 히스토리, 더 높은 편의성에 붙인다.

V1의 첫 유료 모델은 월 구독이 아니라 `분석권 패키지`로 시작한다. 사용자는 연애 상황이 생겼을 때 집중적으로 앱을 쓸 가능성이 크므로, 초반에는 정기 결제보다 필요한 만큼 사서 쓰는 구조가 더 자연스럽다.

결제 제공자는 `Polar`를 사용한다. 결제 완료는 Polar webhook의 `order.paid` 이벤트를 기준으로 처리하고, Cloudflare Workers가 D1의 분석권 잔액을 증가시킨다.

## 기본 원칙

- 무료 사용자는 최소 1번 이상 `아, 이거 쓸 만하다`를 느껴야 한다.
- 유료 전환은 불안감을 자극하기보다 반복 사용과 개인화 가치로 설득한다.
- 민감한 연애 상황을 과금 압박으로 이용하지 않는다.
- 안전 경고, 거절 존중, 개인정보 안내는 무료와 유료 모두 제공한다.
- 유료 기능은 결과 품질을 높이되, 무료 사용자를 일부러 위험하게 만들면 안 된다.

## 무료 제공 범위

V1 무료안:

- 하루 3회 메시지 분석
- 답장 후보 3개 제공
- 현재 분위기 분석
- 기본 이상형/연애 스타일 적합도
- 현재 대화 기준 말투 자동 분석
- 보내면 위험한 말 1개 제공
- 다음 행동 추천

무료에서 제한할 것:

- 분석 히스토리 장기 저장
- 말투 프로필 여러 개 저장
- 이상형 프로필 여러 개 저장
- 상대별 맞춤 코칭
- 데이트 약속 잡기 전용 모드
- 실수 만회 전용 모드
- 고급 패턴 분석

## 유료 제공 범위

V1 유료안:

- 분석권 기반 추가 메시지 분석
- 말투 프로필 저장
- 이상형/연애 스타일 프로필 저장
- 상대별 대화 히스토리 기반 코칭
- 답장 재생성 횟수 확대
- 답장 톤 세부 조정
- AI 티 나는 문장 자동 제거
- 대화 목적별 전용 모드
- 고급 분석: 상대 관심사, 대화 패턴, 다음 타이밍

## 무료/유료 구분 기준

| 기준 | 무료 | 유료 |
|---|---|---|
| 분석 횟수 | 하루 3회 | 구매한 분석권만큼 추가 사용 |
| 답장 후보 | 3개 | 3개 + 재생성 확대 |
| 말투 분석 | 현재 대화 기준 | 저장된 말투 프로필 기반 |
| 이상형 분석 | 기본 1개 | 여러 프로필 저장 |
| 히스토리 | 저장 안 함 또는 짧게 | 상대별 히스토리 |
| 조언 수위 | 기본 3단계 | 상황별 세부 조정 |
| 전용 모드 | 없음 | 약속 잡기, 실수 만회, 관계 회복 |

## 핵심 제품 지표

### Activation Rate (첫 가치 경험률)

첫 사용자가 핵심 가치를 경험했는지 본다.

- 정의: 가입 또는 첫 방문 후 24시간 안에 `분석 결과 도달`과 `답장 복사`를 모두 완료한 사용자 비율
- 공식: `24시간 내 분석 후 답장 복사 사용자 / 신규 사용자`
- 목표 초안: 35% 이상
- 데이터 소스: 이벤트 로그

### Copy Rate (답장 복사율)

AI 답장이 실제로 쓸 만한지 본다.

- 정의: 분석 결과 화면에서 답장 후보를 복사한 비율
- 공식: `답장 복사 세션 / 분석 결과 도달 세션`
- 목표 초안: 40% 이상
- 데이터 소스: 이벤트 로그

### Regenerate Rate (답장 재생성률)

첫 답장이 사용자의 말투나 상황에 맞았는지 본다.

- 정의: 답장 후보를 다시 생성한 비율
- 공식: `재생성 클릭 세션 / 분석 결과 도달 세션`
- 해석: 너무 낮으면 선택지가 부족하지 않을 수 있고, 너무 높으면 첫 결과 품질이 낮을 수 있다.
- 목표 초안: 15~35%
- 데이터 소스: 이벤트 로그

### Tone Match Satisfaction (말투 일치 만족도)

사용자 말투 반영이 잘 되는지 본다.

- 정의: `내 말투랑 비슷함` 피드백 비율
- 공식: `말투 긍정 피드백 / 말투 피드백 전체`
- 목표 초안: 60% 이상
- 데이터 소스: 결과 화면 피드백

### Style Fit Usefulness (스타일 적합도 도움률)

이상형/연애 스타일 적합도 분석이 도움이 되는지 본다.

- 정의: 적합도 카드에서 `도움 됨`을 누른 비율
- 공식: `적합도 도움 됨 / 적합도 피드백 전체`
- 목표 초안: 55% 이상
- 데이터 소스: 결과 화면 피드백

### Free Limit Hit Rate (무료 한도 도달률)

무료 제한이 너무 빡빡한지, 유료 전환 기회가 있는지 본다.

- 정의: 하루 무료 분석 3회를 모두 사용한 무료 사용자 비율
- 공식: `하루 3회 분석 도달 무료 사용자 / 일간 활성 무료 사용자`
- 목표 초안: 10~25%
- 데이터 소스: 사용량 로그

### Paywall Click Rate (유료 안내 클릭률)

유료 안내가 관심을 만드는지 본다.

- 정의: 유료 안내 화면에서 결제 버튼을 클릭한 비율
- 공식: `결제 버튼 클릭 / 유료 안내 화면 노출`
- 목표 초안: 5~12%
- 데이터 소스: 결제 퍼널 로그

### Credit Purchase Start Rate (분석권 구매 시작률)

무료 사용자가 분석권 구매를 시작하는지 본다.

- 정의: 유료 안내 화면 노출 후 분석권 구매 화면으로 이동한 비율
- 공식: `분석권 구매 화면 진입 / 유료 안내 화면 노출`
- 목표 초안: 3~8%
- 데이터 소스: 결제 퍼널 로그

### Credit Purchase Rate (분석권 구매 전환율)

실제 분석권 구매 전환을 본다.

- 정의: 무료 사용자 중 분석권 결제를 완료한 비율
- 공식: `분석권 결제 완료 사용자 / 무료 활성 사용자`
- 목표 초안: 2~5%
- 데이터 소스: 결제 로그

### Repeat Purchase Rate (반복 구매율)

분석권 패키지가 일회성 결제에 그치는지, 반복 구매로 이어지는지 본다.

- 정의: 첫 분석권 구매 후 30일 안에 추가 분석권을 구매한 사용자 비율
- 공식: `30일 내 재구매 사용자 / 첫 분석권 구매 사용자`
- 목표 초안: 15~25%
- 데이터 소스: 결제 로그

### 7-Day Retention (7일 재사용률)

이 앱이 한 번 쓰고 끝나는지, 반복 사용되는지 본다.

- 정의: 첫 분석 후 7일 안에 다시 분석한 사용자 비율
- 공식: `7일 내 재분석 사용자 / 첫 분석 사용자`
- 목표 초안: 20~30%
- 데이터 소스: 이벤트 로그

## 유료 전환 신호

유료 안내를 보여주기 좋은 순간:

- 하루 무료 분석 3회를 모두 사용했을 때
- 같은 상대에 대해 2회 이상 분석했을 때
- 답장을 복사한 뒤 재생성을 여러 번 눌렀을 때
- 말투 프로필을 저장하려고 할 때
- 상대별 히스토리를 보고 싶어 할 때
- 약속 잡기, 실수 만회, 관계 회복 같은 전용 모드에 진입할 때

피해야 할 순간:

- 사용자가 불안하거나 위험한 상황에 있을 때
- 상대가 불편해하는 신호가 강할 때
- 사과나 한 템포 쉬기가 필요한 상황에서 과금 유도를 하는 것
- 개인정보 삭제 안내가 필요한 순간

## Polar 결제 처리 지표

### Webhook Success Rate (웹훅 처리 성공률)

Polar 결제 완료 이벤트가 정상적으로 처리되는지 본다.

- 정의: Polar webhook 수신 후 2xx 응답으로 처리된 비율
- 공식: `성공 처리 webhook / 전체 Polar webhook`
- 목표 초안: 99% 이상
- 데이터 소스: Workers 로그, payment_events 테이블

### Credit Grant Success Rate (분석권 지급 성공률)

결제 완료 후 분석권이 정상 지급되는지 본다.

- 정의: `order.paid` 이벤트 중 D1 분석권 잔액 증가까지 완료된 비율
- 공식: `분석권 지급 완료 이벤트 / order.paid 이벤트`
- 목표 초안: 99% 이상
- 데이터 소스: payment_events 테이블

### Duplicate Webhook Block Rate (중복 웹훅 차단률)

중복 webhook으로 분석권이 중복 지급되지 않는지 본다.

- 정의: 이미 처리된 Polar event id가 다시 들어와 차단된 비율
- 공식: `중복 차단 이벤트 / 전체 Polar webhook`
- 목표 초안: 수치 목표보다 중복 지급 0건이 중요
- 데이터 소스: payment_events 테이블

## 가격 실험 초안

초기에는 분석권 패키지를 먼저 실험하고, 월 구독은 반복 사용 데이터가 쌓인 뒤 검토한다.

우선 실험:

- 분석권 패키지: 30회 3,900원
- 분석권 패키지: 50회 5,900원
- 분석권 패키지: 100회 9,900원

후순위 실험:

- 월 구독: 4,900원
- 월 구독: 7,900원

판단 기준:

- 가끔 급할 때 쓰는 패턴이면 분석권 패키지를 유지한다.
- 반복 구매율이 높고 주간 사용 빈도가 안정적이면 월 구독을 추가 실험한다.
- 말투/상대별 히스토리 사용률이 높으면 구독 가치가 있다.
